How to use What If Studeos

Idea → script → finished vertical video → posted. The full loop, step by step.

1Start everything with one click

Start-What-If-Studio.bat 🎛 Dashboard 👁 Watcher ✦ Studio opens

One double-click starts the whole studio — safe to click again anytime.

  • Open the project folder and double-click Start-What-If-Studio.bat.
  • It starts the dashboard (Videos + Produce pages + the AI writer), the watcher (auto-renders your exports), and opens the Studio in your browser.
  • Tip: right-click the file → Send to → Desktop for a one-click desktop shortcut.

2Pick an idea — or write your own

🎬 Generate + Export

Every scenario becomes a complete production package: hooks, script beats, shot list, captions, thumbnail text.

  • Click any scenario card in the library (search or filter by category).
  • Your own idea? + Create your own scenario → type just the title → ✨ Write it for me drafts the premise and beats (everything stays editable). Pick the video length right next to the button ("for a 90s video") — longer runtimes get longer beats, which means more narration and a longer video. The choice syncs with Package Settings both ways. With an OpenAI key in pipeline\openai_key.txt the draft comes from OpenAI in a couple of seconds; without one it uses the free writer.
  • Choose runtime and voice style — the voice style picks the narrator (Calm=Adam, High-Energy=Callum, Deadpan=George with ElevenLabs). No platform toggle: one cut works on TikTok, Shorts, and Reels, and each finished video's post kit carries the right hashtags for all three.

3One click — the video makes itself

🎬 Generate + Export 📄 📁 Rendering 1 video…

The watcher spots your export and renders it automatically — free AI images, voice, captions, music, covers.

  • Click 🎬 Generate + Export for render — it builds the package and downloads the render .json in one step. The watcher takes it from there.
  • Skim the Script Beats tab while it renders — that's exactly what the voice says. Edited something? Export .json (render again) sends a new cut.
  • A few minutes later the output folder opens with the finished video. New scenarios take 2–5 minutes each (AI images generate once, then they're cached).

4Premium videos — the Produce page

Copy prompt tryinfer Studio ⬇⬇⬇ Import from Downloads ✓ Done

Real AI motion video per beat, using your unlimited tryinfer Studio subscription — no renaming, no folder hunting. With an OpenAI key in pipeline\openai_key.txt, every prompt is automatically rewritten into a richer cinematic direction (once per scenario, then cached).

  • Sidebar → 🎞️ Produce → pick your package from the dropdown. The dropdown is your export history: every export is archived permanently (pipeline\exports), listed newest-first with its date and how many clips you've staged for it — nothing gets overwritten anymore.
  • Spot a typo or want a pause? Open ✏️ Script, fix the hook/beats/outro right there (add a — or … where the narrator should breathe) → Save script → render. The voice, captions, and post kit all follow; your staged clips and caches stay connected.
  • Pick your visuals in step 2: ⚡ AI video (tryinfer API — one real motion clip per beat; pick the model: seedance-2.0-pro is the premium default at ≈$0.65 per 5s clip, while fast/flash models like seedance-2.0-fast or wan2.2-flash cost far less. tryinfer doesn't publish rates, so each model's price appears in the dropdown after your first render with it — learned from your own billed charges via the Spend ledger; cached clips are free), 🖼️ AI images (one styled picture per beat with gentle Ken Burns motion — engine Free Pollinations or any tryinfer image model like nano-banana or flux-2, billed per image; choose Cinematic, 3D, Infographic, or Dark), or 🎬 My own clips.
  • Manual path (uses your Studio sub instead of credit): Copy each numbered prompt → paste into tryinfer Studio (9:16 · 5s · your model) → generate → download in order → ⬇ Import newest videos from Downloads. Hover a clip to preview; ↑↓ to reorder.
  • Don't like a prompt? Rewrite it. Click any numbered prompt, type your own scene, then Save prompts — the Copy buttons and every AI mode (video and images) use your wording from then on, and the style tail (cinematic, 9:16…) is re-added automatically so you never have to type it. Editing the script resets the prompts to match the new beats. Each row also shows the 🎙 spoken line (hook / beat / outro) the narrator says over that clip, so you always know which words go with which visual — and it's editable right there: click the line, type, then ✓ save (or ↩ undo). Click ✨ and the AI punches it up (same facts, same length) — and when that beat has a selected image or video, the AI looks at it and rewrites the line to match what's on screen (for videos it reads the first frame). ✓ keep writes it into the script (voice, captions, and post kit follow; your visual prompts are kept), ↩ undo throws it away. Uses your OpenAI key when present, otherwise the free writer (text-only).
  • Nothing saved is ever lost. Every beat row has its own 🕘 History button: past versions of that beat's spoken line and prompt, with Use to bring one back (it lands as a proposal — keep / Save prompts to apply). The 🕘 Full history panel below the prompts lists whole saved versions of the script and prompts (the very first save also archives the original), and Restore brings any of them back — restoring is recorded too, so you can even revert a revert. Note the flip side: an auto-described prompt or ✨ suggestion you haven't saved/kept disappears on refresh — images and videos, by contrast, are always saved the moment you attach them.
  • Or start from a picture — or your own clip. Every prompt row has an ＋ image and an ＋ video box (also: drag a file onto the row, or point at it and paste Ctrl+V for pictures). Both are saved with the package (click to replace, ✕ to remove), and the radio button next to them picks which one the beat uses. Attaching an image auto-describes it into the prompt (so frame and motion always agree — undo or retype if you prefer your own words); 🖼 From image re-runs it anytime, and 🎬 From video does the same from your clip's first frame. All need the OpenAI key; each costs a fraction of a cent. Tweak, then Save prompts.
  • Radio on the image: in ⚡ AI video mode that beat is animated from your picture — cropped to 9:16 and sent as the clip's first frame, so the video literally starts on your photo (beats without one keep the auto-generated first frame). Swap the image and only that beat regenerates on the next render; the rest stay cached and free.
  • Radio on the video: that beat plays your uploaded clip as-is — in every mode, including ⚡ AI video, where it also skips that beat's generation cost entirely. Mix freely: some beats from images, some from your own clips, the rest AI-generated. Add Keep clip sound at render time if your clip's audio should stay under the voiceover.
  • Tick ElevenLabs voice, pick a narrator from the Voice dropdown — click ▶ Hear it to play a short sample of any voice (free) before you commit. Auto shows who the package's style would pick. Add Keep clip sound if you used LTX → Render → go do something else: when it finishes you get a desktop notification, a chime, and the tab title flips to ✅ DONE (allow notifications when the browser asks, the first time you render).

5Review your pipeline

notes autosave…

The Videos page is your posting queue — order it the way you plan to post.

  • Sidebar → 🎬 Videos. Watch each video right on the page.
  • ↑ ↓ set your posting order (it's saved). #1 = post next.
  • Write notes per video (hook ideas, when to post) — they autosave.
  • Remove moves a video to output/trash — recoverable, never deleted.
  • Sidebar → 💰 Spend tracks every metered API charge: tryinfer clip/image prices are the provider's exact billed amounts; OpenAI drafts and prompt polish are token-count estimates (fractions of a cent). Flat subs (ElevenLabs, tryinfer Studio) aren't metered.
  • The OpenAI · this month card can show your real billed OpenAI cost: create an Admin key at platform.openai.com (Settings → Admin keys) and drop it in pipeline\openai_admin_key.txt. Your remaining balance is something OpenAI only shows in their own dashboard — the card links straight to it.

6Post it (the part only you can do)

⬆ ✓ Caption + hashtags from the post kit ✓ Cover image (-thumb.jpg) ✓ "AI-generated content" toggle ON

Everything you need to paste is in the video's post kit — posting takes about two minutes.

  • Watch the video once — you're the editor of record.
  • Upload the .mp4, set the matching -thumb.jpg as the cover.
  • Open the post kit (on the video's card) → paste a caption + the music credit. Every kit now includes a cross-posting block with the right hashtags for TikTok, YT Shorts, and Reels — same video, swap the tags.
  • Turn on the platform's AI-generated content disclosure → post.

?When something's off

"Dashboard offline" or the AI writer won't work

The helper isn't running. Double-click Start-What-If-Studio.bat in the project folder, then try again.

"The writing service hiccuped"

The free AI writer handles one request at a time — wait a minute and click ✨ Write it for me again. To skip the line, paste an OpenAI API key into pipeline\openai_key.txt (it works on the very next click — no restart): drafts then come from OpenAI (a fraction of a cent each) and automatically fall back to the free writer if the key stops working.

I want different images for a video

Delete that scenario's folder inside pipeline\ai-visuals — the next render generates a fresh set (now with people in frame by default). For fresh prompt wording too, also delete the scenario's file in pipeline\polished-prompts.

My custom scenarios disappeared / "Memory only" badge

Custom scenarios save in this browser on this device. If the badge says Memory only, the browser is blocking storage for local files — export anything you want to keep, and use the same browser each time. (Your videos and exports are files on disk and are never affected.)

Where do my videos live on disk?

pipeline\output — each video with its cover image (-thumb.jpg) and post kit (-post.txt). Removed videos are in pipeline\output\trash.
